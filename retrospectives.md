#  Sprint 1 Retrospective  
**โปรเจกต์:** TrailTeller  
**ระยะเวลา:** ตุลาคม 2025  
**ทีม:**  
**เป้าหมายของ Sprint:** วางแนวทางโครงสร้างระบบ TrailTeller ทั้งฝั่งเว็บและโมบายล์ รวมถึงร่างแผนงานสำหรับ Backend ที่จะพัฒนาใน Sprint ถัดไป

---

##  สิ่งที่ทำใน Sprint นี้ (What We Did)
- วางโครงสร้างโปรเจกต์ **Frontend (React + V0 + ChatGPT,)**  
  - ออกแบบหน้า **Login, Register,Planner,Recommendation,Profile,Budget**  
  - ใช้ **V0 (AI UI Generator)** เพื่อสร้างและปรับแต่ง UI ได้รวดเร็ว  
  - ตั้งค่า  component พื้นฐาน
- วางแนวคิดและร่างโครงสร้าง **Backend (ยังไม่พัฒนา)**  
  - วางแผน API สำหรับ Register, Login, และ User Profile   
- สร้าง repository สำหรับ frontend และ backend แยกกันบน GitHub   
- เริ่มต้นสร้าง **Mobile App (React Native)** พร้อมการ setup เบื้องต้น

---

##  สิ่งที่ทำได้ดี (What Went Well)
- การใช้ **V0** และ **AI Tools (เช่น ChatGPT)** ช่วยสร้างและแก้ไข UI ได้อย่างรวดเร็ว   
- การจัดการ repository  ทำให้เห็นภาพรวมของโครงงานได้ง่าย  
- โครงสร้าง Frontend พร้อมรองรับการเชื่อมต่อ Backend ใน Sprint ถัดไป  

---

##  สิ่งที่ไม่ราบรื่น (What Didn’t Go Well)
- **Backend ยังไม่ถูกพัฒนาและไม่เชื่อมต่อกับ Frontend**  
- บางส่วนของ UI ยังไม่สมบูรณ์และต้องแก้ไข layout เพิ่มเติม  
- ยังไม่มีระบบจัดการ state หรือ data flow ที่ชัดเจน  
- การจัดลำดับความสำคัญของ task ยังไม่เป็นระบบพอ  

---

##  สิ่งที่ควรปรับปรุง (What Could We Improve)
- เริ่มพัฒนา Backend จริงใน Sprint 2  
- ออกแบบและทดสอบ API ด้วย Postman ก่อนเชื่อมกับ Frontend  
- สร้างเอกสาร API และกำหนดรูปแบบ response ให้ชัดเจน  
- ปรับปรุงการจัดการ branch และ commit ให้เป็นระเบียบ 


---

###  สรุป
Sprint 1 เป็นช่วงของการวางรากฐานโครงสร้างโปรเจกต์ TrailTeller โดยเน้นการเตรียม Frontend และวางแผนสำหรับ Backend ที่จะพัฒนาใน Sprint ถัดไป  
แม้ยังไม่มีการเชื่อมต่อระบบ แต่ทีมสามารถวางแนวทางได้ชัดเจนและพร้อมสำหรับการเริ่มพัฒนา Backend ใน Sprint 2
